# Issue 005: Comparison Readiness Matrix

## Parent

health-004

## What To Build

Create a comparison-readiness matrix for normalized fields the navigator can
compare safely across PhilHealth, HMOs, and insurer health-protection products.
The matrix is not a recommendation engine.

## Acceptance Criteria

- [x] Matrix defines comparable dimensions such as entity type, market segment,
      product type, source authority, benefit category, network verification,
      extraction status, and human-review need.
- [x] Matrix separates comparable normalized fields from non-comparable prose.
- [x] Matrix blocks ranking, scoring, recommendation, coverage decisioning,
      claims decisioning, and eligibility decisioning.
- [x] Matrix references existing registry and product catalog artifacts.
- [x] Validator checks that blocked outputs remain explicit.

## Blocked By

None. Registry and product catalog artifacts already exist.
